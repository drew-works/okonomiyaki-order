
# お好み焼き屋さん 公式ホームページ
私がアルバイトをしている実際の店舗（お好み焼き・もんじゃ焼き店「ごきらく亭」）のオーナーからヒアリングを行い、実用性を重視して制作した公式ホームページです。

## 🔗 サイトURL
https://drew-works.github.io/okonomiyaki-order/HP/

## ✨ 本プロジェクトにおけるこだわり・開発プロセス

### 1. 徹底したユーザー目線のUX向上と課題解決
- **エンドユーザー（店舗のお客様）の体験（UX）最大化：** スマートフォンでの閲覧を第一に想定し、CSSメディアクエリを用いた徹底的なレスポンシブ対応を施しました。キャッチコピーが画面幅で不自然に改行されるのを防ぐため、`white-space: nowrap` と独自の改行コントロール（`<br class="sp-only">`）を組み合わせ、ファーストビューの「アツアツ感」を視覚的に損なわない工夫をしています。
- **ヒアリングに基づく店舗課題の解決：** オーナーとの対話から、ワンタップで発信できる「電話予約専用セクション」の設置や、事前のお客様の不安（服の匂い、決済方法など）を解消するための「よくあるご質問（Q&A）」を実装し、店舗と顧客の双方にメリットがある情報設計を行いました。

### 2. 「AIとの共創」による開発速度の最大化と拡張性の担保
- **AI（LLM）を開発パートナーとした設計の言語化：**
  本プロジェクトでは、単にコードを自動生成させるのではなく、「店舗オーナーからの定性的な要望」をAIと共にブレインストーミングし、最適なデータ構造やマークアップ（HTML/CSS）へと落とし込む協働開発を実践しました。
- **拡張性と保守性を意識したコード設計：**
  公開後、オーナーからのメニューについての細かい仕様変更の要望に対し、CSSのクラス名とHTMLのカード構造をシンプルに紐付ける設計をAIと共創。これにより、今後のメニュー変更にも即座に対応できる高い保守性を実現しました。また、価格改定に伴い、スマートフォンの限られた画面幅でも視認性を保てるよう、税込表記を薄くコントロールする専用スタイル（`tax-included`）への仕様改善を迅速にデプロイしました。

### 3. 自力でのトラブルシューティングと継続的改善
- **課題解決力：**
  GitHub Pagesを活用した本番環境へのデプロイにおいて、Git push時のエラーやバッファサイズの問題が発生した際も、ログを解析し自力で原因を特定して一般公開までやり遂げました。
- **アジャイルな仕様変更への対応：**
  「作って終わり」ではなく、オーナーからの要望を迅速に反映させるため、Gitを用いた確実なバージョン管理と、継続的な機能改善のサイクルを回し続けています。
